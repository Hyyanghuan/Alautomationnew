# AI自动化测试平台 V3.0 - 使用手册

## 一、快速开始

### 默认账号

| 字段 | 值（可在 .env 修改） |
|------|---------------------|
| 用户名 | `admin` |
| 密码 | `admin123` |
| 角色 | 超级管理员 |

登录后请立即修改密码（通过用户管理扩展或数据库）。

### 核心流程

```
创建项目 → 添加版本/功能 → AI生成测试点 → 编辑保存 → 生成用例 → 创建计划 → 执行 → 查看报告
```

## 二、功能模块使用

### 2.1 项目管理与各独立页面

**项目管理**（`/projects`）仅维护项目基本信息。其余模块在左侧菜单**单独进入**，通过页顶 **「关联项目」** 下拉框从已创建项目中选择（按创建时间倒序，每页 10 条，超过 10 个时下拉滚动自动加载更多；支持搜索过滤），选择会记忆上次关联。

| 菜单 | 路径 | 说明 |
|------|------|------|
| 项目管理 | `/projects` | 新建/编辑项目，状态与计划时间管理 |
| 版本管理 | `/versions` | 需关联项目；启动/挂起/完成/编辑 |
| 测试点 | `/test-points` | 需关联项目；需求文档卡片选择 |
| 功能管理 | `/features` | 在测试点下方；由测试点自动同步 |
| 用例管理 | `/test-cases` | 需关联项目；下拉选择计划关联 |
| 测试计划 | `/test-plans` | 需关联项目；显示关联用例数 |
| 测试执行 | `/executions` | 执行计划与历史记录 |
| 测试报告 | `/reports` | 可选项目筛选 |
| 执行器规则 | `/executor-rules` | 查看执行器代码规则 |

#### 项目管理操作

| 操作 | 说明 | 适用状态 |
|------|------|----------|
| **启动** | 项目进入进行中 | 规划中、已暂停、已挂起 |
| **暂停** | 临时停止推进 | 进行中 |
| **挂起** | 长期搁置 | 进行中、已暂停 |
| **完成** | 项目结案 | 规划中/进行中/已暂停/已挂起 |
| **重启** | 重新进入进行中 | 已暂停、已挂起、已完成 |
| **编辑** | 修改名称、描述、开始/结束时间 | 未完成且未归档 |

关联关系：版本/功能/测试点/用例归属项目；用例与计划为多对多关联；执行记录关联计划与项目。各页面通过 **关联项目** 选择器绑定项目（记忆上次选择）。

#### 完整测试流程（推荐顺序）

```
1. 项目管理 → 创建并启动项目
2. 版本管理 → 添加版本
3. 测试点   → 上传需求（卡片选择）→ Agent 生成测试点 → 保存
4. 功能管理 → 自动从测试点同步（可手动「从测试点同步」）
5. 用例管理 → Agent 生成用例 → 下拉选择计划关联
6. 测试计划 → 确认关联用例数 > 0
7. 测试执行 → 执行计划 → 查看结果/日志
8. 测试报告 → 查看通过率与明细
```

### 2.2 需求文档上传（测试点生成前置）

路径：项目 → **测试点** → **需求文档** 标签

| 方式 | 支持格式 |
|------|----------|
| 文本录入 | 直接粘贴 text |
| URL 抓取 | 在线需求链接 |
| 文件上传 | pdf、word(doc/docx)、md、txt |

上传后点选需求文档卡片，在页顶 **生成测试点树** 栏选择 **测试设计 Agent**（可切换不同模型），点击「Agent 生成测试点树」；生成完成后自动切换到测试点树标签。

### 2.3 测试点树（XMind 式）

| 操作 | 说明 |
|------|------|
| 选择 Agent | 同类型（design）可切换不同模型实例，如 Qwen-Plus / GPT-4o / DeepSeek |
| 引用知识库 | 多选全局或项目知识库，注入 Agent 上下文 |
| Agent 生成测试点 | 基于需求文档 + 功能列表 + 知识库，由测试设计 Agent 生成 JSON 树 |
| 拖拽编辑 | 在树组件中调整节点结构 |
| 保存 | 持久化测试点树并同步功能列表 |
| Agent 生成用例 | 勾选叶子节点生成用例（自动先保存测试点树） |

### 2.4 用例管理

路径：**用例管理**

- 分页、关键词搜索、按测试类型筛选
- **关联计划**：单条/批量通过下拉选择测试计划（无需手输 UUID）
- **Agent 核查**：选择测试设计 Agent（可换模型），勾选后用例质量评分与建议
- 预置测试类型：功能、接口、性能、安全、UI、冒烟、回归、Agent测试 等

### 2.5 测试计划

路径：**测试计划**

- 创建计划后，在 **用例管理** 中将用例关联到计划
- 列表显示 **关联用例数**；为 0 时不可执行
- 执行后跳转测试报告

### 2.6 测试执行

路径：**测试执行**（`/executions`）

| 标签 | 功能 |
|------|------|
| 执行计划 | 按计划执行，显示用例数 |
| 执行记录 | 历史记录，跳转结果/日志/报告 |

**执行器规则**（`/executor-rules`）：查看各执行器说明与代码示例。

脚本类用例执行失败时，自动调用 **自愈修复 Agent**（healing 类型）分析日志并尝试修复重试。

子页面：`/executions/{id}/results`、`/executions/{id}/logs`

### 2.7 Agent 智能体管理（完整文件结构）

路径：**Agent管理**

启动后已预置 4 个全局智能体，每个含完整 **Agent 文件结构**（与标准目录一致）：

| 目录 | 内容 |
|------|------|
| prompts/ | system-prompt、task-router、role-config、output-format、task-prompts |
| memory/ | 用户画像、会话上下文、长期记忆 |
| tools/ | 浏览器、代码执行、API、检索 |
| workflows/ | 计划执行、RAG、多 Agent、评估闭环 |
| skills/ | 写作、编码、分析、设计 |
| knowledge/ | FAQ、产品文档、策略、术语表 |
| configs/ | model-settings、tool-permissions 等 |

点击 **配置详情** 可在前端树形浏览并编辑各文件，保存后写入 `PUT /api/v1/agents/{id}/profile`。

**Agent 类型与使用位置：**

| 类型 | 预置实例 | 使用位置 |
|------|----------|----------|
| design | 测试设计智能体、测试设计·GPT-4o、测试设计·DeepSeek | 测试点生成/用例生成/用例核查 |
| healing | 自愈修复智能体 | 测试执行失败自动修复 |
| code_analysis | 代码分析智能体 | Agent 管理（扩展） |
| defect_prediction | 缺陷预测智能体 | Agent 管理（扩展） |

同类型可再创建多个 Agent，绑定不同大模型；业务页面下拉按类型筛选。

### 2.8 AI 模型管理（仅管理员）

路径：**AI模型**

启动后预置：GPT-4o、GPT-4o-Mini、Qwen-Plus、Qwen-Turbo、DeepSeek-Chat、DeepSeek-Reasoner。

| 功能 | 说明 |
|------|------|
| 配置 Key | 每个模型可 **单独配置 API Key**（`PATCH /ai-models/{id}/api-key`） |
| 编辑 | 修改 API 地址、限流等 |
| 测试连接 | 验证 Key 与模型可用性 |

`.env` 中 `OPENAI_API_KEY`、`QWEN_API_KEY`、`DEEPSEEK_API_KEY` 会在首次启动时写入对应预置模型；前端配置优先于 .env 空值补全。

### 2.9 知识库管理（仅管理员）

路径：**知识库**

启动后自动预置 **测试平台规范知识库**（流程、优先级、设计方法、术语表、Agent 说明等 8 条可编辑条目）。

| 操作 | 说明 |
|------|------|
| 新建/编辑/删除 | 知识库基本信息 |
| 管理条目 | 新增、编辑、删除 Markdown 知识条目（policy/design/glossary/faq 等分类） |
| 上传文档 | txt/md 等文本文件自动解析为条目 |
| 检索测试 | 在条目抽屉内验证关键词召回 |
| 测试点引用 | 测试点页多选知识库，Agent 生成时自动注入 Top-K 片段 |

### 2.10 Agent 专项测试

通过 **Agent测试** 类型用例 + **AgentTestExecutor** 执行：

- 意图识别准确率
- 工具调用轨迹
- 多轮对话连续性
- RAG 知识召回准确性

## 四、API 调用示例

### 登录获取 Token

```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

### AI 生成测试点

```bash
curl -X POST http://localhost:8000/api/v1/test-points/generate \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"project_id":"<项目UUID>"}'
```

### 执行测试计划

```bash
curl -X POST http://localhost:8000/api/v1/executions/run/<计划UUID> \
  -H "Authorization: Bearer <token>"
```

完整 API 文档：http://localhost:8000/docs
